# 飞书待办桌面端设计规范

## 设计定位

产品采用“macOS 提醒事项 × 飞书原生信息层级”。常驻面板首先是安静、轻量的系统工具，其次才是任务管理器。界面不使用桌宠、头像、品牌吉祥物、彩色整卡、仪表盘式统计或网页 SaaS 导航。

## 设计原则

1. 内容优先：常驻面板只展示当下需要处理的任务。
2. 系统原生：使用 SF Pro、AppKit 原生窗口行为和 macOS 材质。
3. 单一主色：所有可交互状态使用飞书蓝；系统红只表示逾期或高风险。
4. 渐进披露：完成操作直接可见，其余动作进入详情或更多菜单。
5. 低打扰：默认 360 × 260px，最多展示 3 条任务，不显示统计卡和长期管理信息。

## 设计令牌

### 颜色

| 令牌 | 值 | 用途 |
|---|---|---|
| `canvas` | `#FFFFFF` / 系统窗口材质 | 主背景 |
| `surface-subtle` | `#F5F5F7` | 轻量标签、元信息胶囊 |
| `ink` | `#1D1D1F` | 标题与主要文本 |
| `ink-secondary` | `#646A73` | 次级说明 |
| `ink-tertiary` | `#8E8E93` | 日期、来源、禁用状态 |
| `divider` | `rgba(60,60,67,0.12)` | 任务行分隔线 |
| `feishu-blue` | `#3370FF` | 链接、选中、焦点与主要操作 |
| `danger` | `#F54A45` | 逾期与高风险状态 |

深色模式必须使用对应的 macOS 动态系统色，不直接反转固定十六进制颜色。

### 字体

- 字体族：`SF Pro Text`, `SF Pro Display`, `-apple-system`。
- 页面标题：19px / 600，紧凑字距。
- 任务标题：13px / 400；优先任务可使用 500。
- 分组标题：10px / 600，大写或小型标题样式。
- 日期与元信息：10px / 400。
- 禁止使用全大写英文状态、网页式超粗标题和超过两级的字重竞争。

### 间距与圆角

- 基础间距单位：4px。
- 常用节奏：4 / 8 / 12 / 16 / 24px。
- 面板内边距：16px。
- 任务行高度：36px。
- 任务行间距：0，使用细分隔线区分。
- 小型元信息胶囊圆角：6px。
- 面板圆角：18px。
- 圆形完成控件：14px。

## 常驻面板

### 尺寸与窗口

- 默认尺寸：360 × 260px。
- 最小尺寸：320 × 220px。
- 最大尺寸：560 × 520px。
- 使用透明标题栏、原生交通灯和轻量阴影。
- 展开与折叠必须使用独立尺寸边界，禁止折叠内容停留在展开窗口中。

### 信息结构

1. 标题“今天”。
2. 次级信息：“N 项待办 · 飞书已同步”。
3. 右上角一个新增按钮。
4. “优先处理”分组。
5. “接下来”分组。
6. 总计最多显示 3 条任务。

常驻面板禁止展示：双统计卡、筛选器、排序器、同步工具栏、Base 按钮、任务优先级字母、整卡背景以及每行更多按钮。

### 任务行

- 左侧：圆形完成控件。
- 中间：单行任务标题，超长截断。
- 右侧：截止时间、今天或“飞书”来源标签。
- 点击任务标题进入详情。
- 高风险任务只在元信息或小型状态上使用红色，不染色整行。

## 完整窗口

完整窗口延续相同令牌与组件语法，可展示筛选、排序、Base 状态、全部任务、已完成任务和设置。常驻面板与完整窗口必须看起来属于同一系统，但信息密度不同。

## 交互状态

- Hover：使用 `surface-subtle`，不改变布局。
- Focus：2px 飞书蓝焦点环。
- Pressed：使用系统原生按压反馈。
- Completed：标题降低透明度并显示系统完成标记。
- Offline：只在标题副文案显示“等待飞书同步”，不弹出持续错误横幅。

## 禁止项

- 小狗、桌宠、头像和人格化称呼。
- “Aime/Aily”等用户可见名称，统一称为“飞书机器人”。
- 红、橙、蓝整块任务卡。
- 英文 `open / overdue / focus` 状态。
- 默认 AppKit 灰色按钮堆叠。
- 同一层级同时出现三个以上按钮。
- 用设计风格偏好切换回历史布局。

## 验收标准

- 360 × 260px 下无需滚动即可看到标题、同步状态和 3 条任务。
- 常驻面板内最多只有一个显式新增操作。
- 除飞书蓝和系统红外，不出现其他强调色。
- 任务行不存在整卡填充色与常驻更多按钮。
- 中文、深色模式、键盘焦点和窗口缩放均可用。
