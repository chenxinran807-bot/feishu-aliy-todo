# 神仙待办 Apple 原生视觉统一设计

日期：2026-07-10

## 目标

将常驻小组件、展开面板和完整管理窗口统一为克制的 macOS 原生视觉语言，同时移除固定小狗形象，改为用户可自定义上传的个人头像。

本次改造只调整视觉系统、身份入口和头像本地配置，不改变任务、飞书同步、主动建议与快捷操作的既有业务流程。

## 设计方向

采用 Apple 风格的“原生克制”方向：

- 白色主画布与 `#F5F5F7` 辅助表面。
- 系统蓝作为唯一主操作色。
- 使用系统字体栈，中文优先回退至苹方。
- 使用细分割线、轻量毛玻璃与有限阴影表达层级。
- 常驻小组件保持宽松，展开面板适度紧凑，完整窗口保持清晰而不过度压缩。
- 不引入 Apple 品牌标识、专有图片或容易造成品牌混淆的资产。

## 视觉系统

### 色彩

- 主画布：白色。
- 辅助表面：Apple 式浅灰 `#F5F5F7`。
- 主操作：系统蓝。
- 危险与逾期：系统红，仅用于语义状态。
- 主文字：近黑色；次级文字使用系统灰。
- 边框：低透明度系统灰细线。

### 排版

- 使用 `-apple-system`、`BlinkMacSystemFont`、`SF Pro Text`、`PingFang SC` 等系统字体回退。
- 标题采用 600 字重，不使用过重的 700/800 作为常规层级。
- 正文保持舒适行高；任务列表优先支持扫读。
- 通过字号、字重和留白建立层级，减少彩色标签的使用。

### 形状与层级

- 控件使用 8–12px 圆角。
- 浮层与主要容器使用 18–24px 圆角。
- 头像始终为圆形。
- 任务卡片主要依靠表面色差和细边框分层。
- 阴影仅用于常驻小组件、展开面板和真正悬浮的界面元素。
- 毛玻璃仅用于浮层或侧边栏，不作为普通卡片装饰。

## 三层界面

### 常驻小组件

- 保持低打扰、可长期停留的视觉体量。
- 展示用户头像、今日数量、逾期数量、下一项任务和必要同步状态。
- 头像是身份识别元素，不再承担宠物状态表达。
- 只有浮层外壳使用轻量阴影和半透明背景。

### 展开面板

- 保持适度紧凑，优先展示逾期、今日、稍后任务和主动建议。
- 快捷操作在悬停、聚焦或选择任务后出现，减少常态噪音。
- 头像保持小尺寸身份入口，可进入完整窗口或设置。

### 完整管理窗口

- 侧边栏使用浅灰半透明表面，主内容使用白色画布。
- 分类、搜索、任务列表、设置和同步信息使用统一的控件与间距规则。
- 设置区提供头像预览、更换头像和恢复首字头像。

## 用户头像

### 展示

- 已上传图片按中心裁切，以圆形显示。
- 图片不拉伸，使用等比例覆盖方式填满圆形容器。
- 未上传图片时显示姓名首字母或中文昵称首字。
- 姓名与昵称均为空时显示“神”。

### 上传

- 点击头像或设置区的“更换头像”打开系统文件选择器。
- 支持 PNG、JPEG 和 WebP。
- 选择后显示圆形预览；用户确认后保存。
- 设置区提供删除头像并恢复首字头像的操作。

### 存储与隐私

- 头像原图仅保存在本机应用数据目录。
- 头像不上传飞书或其他远程服务。
- 三层界面读取同一份头像配置，保存后立即刷新。
- 配置模型保存昵称、头像本地路径和回退首字所需信息。

### 异常处理

- 格式不支持、文件损坏或读取失败时保留当前头像。
- 错误提示使用简短、可恢复的本地提示，不阻断其他功能。
- 头像文件被外部删除时自动回退到首字头像。

## 组件与技术边界

- 使用通用 `UserAvatar` 组件替换前端 `DogPortrait`。
- 常驻小组件、展开面板、完整窗口和设置区使用同一头像组件及配置源。
- 新增头像配置的本地读写接口和对应类型。
- 原生 macOS 壳层删除狗图标、狗状态文本和狗图标默认回退。
- 原生壳层与 React 界面读取一致的头像语义：自定义图片优先，首字头像回退。
- 视觉令牌集中定义，组件不得自行引入第二套主色、圆角或阴影体系。

## 测试与验收

### 自动化测试

- 首字生成：英文姓名、中文昵称、空名称。
- 头像回退：无图片、图片不存在、读取失败。
- 文件限制：PNG、JPEG、WebP 可接受，其他格式拒绝。
- 删除头像后恢复首字头像。
- `UserAvatar` 在图片和首字两种状态下正确渲染。
- 设置区包含预览、更换和删除入口。
- 既有任务与飞书同步测试继续通过。
- 原生 Swift 品牌与头像测试不再依赖狗图标。

### 视觉验收

- 常驻小组件、展开面板和完整窗口视觉语言一致。
- 自定义头像在所有入口均为居中圆形裁切。
- 未上传头像时，所有入口显示一致的首字头像。
- 键盘焦点清晰，交互控件具备可见聚焦状态。
- 小尺寸控件不低于 36px 可点击区域，主要触控入口目标为 44px。
- 不出现文本溢出、头像变形、内容遮挡或无意义的重阴影。

## 非目标

- 不重构任务数据模型。
- 不改变飞书同步协议或授权流程。
- 不增加云端头像同步。
- 不增加图片编辑、缩放、旋转或手动裁切工具。
- 不引入深色模式；本轮统一浅色原生体验。
- 不提交 Apple 品牌资产或复制受保护的视觉素材。

## 完成标准

- 三层界面均采用统一的 Apple 原生浅色视觉语言。
- 项目界面和原生壳层不再出现固定小狗图标或相关宠物文案。
- 用户能够上传、预览、更换和删除本地头像。
- 头像按中心裁切并显示为圆形。
- 无头像时稳定回退至姓名首字母、昵称首字或“神”。
- 自动化测试、生产构建和原生测试通过。
