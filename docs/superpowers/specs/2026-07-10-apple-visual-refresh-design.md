# macOS × 飞书轻量待办面板设计

日期：2026-07-12

## 目标

将桌面常驻层改造成低打扰、可自由缩放的轻量任务面板。视觉采用 macOS 原生窗口质感与飞书原生色彩、同步和协作语义。

## 最终范围

- 完全删除小狗、桌宠、宠物状态、喂养、奖励和陪伴型文案。
- 不展示用户头像、昵称首字、产品名称或品牌区。
- 默认面板尺寸为 `360 × 260px`。
- 支持拖拽窗口边缘缩放，范围为 `320 × 220px` 至 `560 × 520px`。
- 记住用户上次设置的窗口尺寸和位置。
- 提供小、标准、大三个预设：`320 × 220`、`360 × 260`、`480 × 420`。
- 完整管理窗口继续按需打开，不长期占用桌面。

## 视觉语言

- macOS：系统字体、圆角窗口、交通灯、浅色毛玻璃、轻量浮层阴影和清晰焦点态。
- 飞书：主操作色 `#3370FF`、危险色 `#F54A45`、正文 `#1F2329`、次级文字 `#646A73`、浅灰表面 `#F5F6F7`。
- 常驻面板不显示侧边栏，只显示日期、飞书同步状态、今日/逾期数量和当前任务。
- 任务来源、同步状态与协作信息可见，但不使用品牌 Logo。
- 建议只作为低干扰灰蓝提示，不抢占任务主层级。

## 自适应规则

- 小尺寸：展示概览和最多 3 条任务，隐藏建议、进度和次要操作。
- 标准尺寸：展示概览、3–5 条任务与简短同步状态。
- 大尺寸：展示更多任务、建议和进度，但仍不显示永久侧边栏。
- 内容按容器宽高自动切换，不通过缩放字体来硬塞内容。
- 主要点击目标不小于 36px，键盘焦点始终可见。

## 技术边界

- 在 `CompanionSettings` 中保存窗口宽高。
- Web 预览使用响应式容器查询或媒体规则。
- 原生 AppKit 窗口设置最小/最大尺寸，监听窗口缩放并持久化。
- 删除 React 与 Swift 中所有固定狗图标和宠物化默认文本。
- 不改变飞书同步协议、任务数据模型或既有任务操作。

## 验收标准

- 默认启动尺寸为 `360 × 260px`。
- 用户可在限制范围内自由调整大小，重启后尺寸与位置恢复。
- 小/标准/大预设可用。
- 项目界面和原生壳层不出现用户头像、产品品牌区、小狗或桌宠文案。
- 三个尺寸下无文本溢出、控件遮挡或不可操作内容。
- 自动化测试、TypeScript 检查、生产构建和 Swift 测试通过。
