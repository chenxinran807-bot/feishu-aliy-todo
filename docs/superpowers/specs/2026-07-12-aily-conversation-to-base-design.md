# 飞书 Aily 对话建待办设计

日期：2026-07-12

## 目标

用户只需与飞书 Aily 对话即可创建待办。Aily 负责从明确指令、聊天、会议或文档中提取任务并写入专用 Base；桌面面板负责自动展示、完成和改期。Base 是长期记录与唯一任务事实源，用户日常不需要直接操作 Base。

## 核心链路

```text
用户与飞书 Aily 对话
→ Aily 识别明确待办或总结候选待办
→ 明确指令直接写入；总结结果等待用户确认
→ Aily 调用 Base 新增记录
→ 通知桌面端增量同步
→ 桌面面板展示新待办
```

### 明确指令

例如：“穿搭 Tab 实验分析明天截止。”

- Aily 解析标题和相对时间。
- 相对时间转换为绝对日期时间后写入 Base。
- 写入成功后回复创建结果。
- 写入失败时明确回复失败，不伪装成功。

### 内容总结

例如：“读取这段聊天，总结待办。”

- Aily 返回候选待办清单。
- 用户确认后批量写入 Base。
- 未指定截止时间的任务允许创建，截止时间留空。
- 批量写入允许部分成功；失败项可单独重试。

## Aily 回复

明确指令成功：

> 已创建「穿搭 Tab 实验分析」，截止时间：明天 18:00。

总结确认前：

> 从内容中识别出 3 个待办：
> 1. 完成实验数据分析——明天
> 2. 整理评审材料——周五
> 3. 跟进研发排期——未指定时间
> 确认后将写入待办库。

确认写入后，Aily 回复成功数量并列出失败项。

## Base 数据模型

| 字段 | 用途 |
|---|---|
| 待办标题 | 桌面面板主标题 |
| 状态 | 待处理、进行中、已完成、已忽略 |
| 截止时间 | 支持为空 |
| 创建时间 | 自动记录 |
| 更新时间 | 增量同步依据 |
| 创建人 | 谁通过 Aily 创建 |
| 来源类型 | 私聊、群聊、会议、文档、手动 |
| 来源链接 | 回到原始上下文 |
| 原文摘要 | 保留创建依据 |
| 幂等 ID | 防止重复创建 |
| 解析置信度 | 排查错误与优化提取策略 |

Base 记录 ID 是跨端任务身份。桌面端不得为同一任务生成另一套远程身份。

## 确认策略

- 单条明确指令：直接写入。
- 聊天、会议、文档总结：用户确认后批量写入。
- 信息不完整：允许创建并保留空字段，不擅自推断。
- 用户可在确认前编辑标题和截止时间。

## 同步机制

采用通知触发与定时轮询结合：

1. Aily 成功写入 Base 后发送同步通知。
2. 桌面端收到通知后，按 Base 记录 ID 或更新时间增量拉取。
3. 通知未送达时，保留 5 分钟轮询兜底。
4. 桌面端恢复网络或重新启动后补拉遗漏记录。

正常情况下，Aily 创建后的任务应在 3 秒内出现在桌面面板；通知链路失败时，最晚 5 分钟出现。

## 去重

- Aily 为每条候选任务生成稳定幂等 ID。
- 同一批总结重复确认不创建重复记录。
- 相同来源、标题和截止时间的任务视为疑似重复，提示用户确认。
- 用户明确要求“再创建一个”时允许覆盖去重。

## 桌面写回

- 完成和改期写回同一条 Base 记录。
- 成功后重新增量拉取确认远程状态。
- 写回失败时显示“待同步”，保留操作并后台重试。
- 不得在远程写回失败时向用户展示已同步成功。

## 错误与恢复

- Aily 写入失败：保留候选内容并提供重试。
- 批量部分失败：成功项不回滚，失败项单独重试。
- 桌面端离线：Base 继续作为事实源，恢复后补拉。
- 重复通知：幂等处理，不重复刷新或创建任务。
- 无法解析相对时间：要求用户确认，不自行猜测。

## 非目标

- 不要求用户直接编辑 Base。
- 不让桌面端承担自然语言任务提取。
- 不维护第二套云端任务数据库。
- 不自动发送外部消息或替用户完成任务。
- 本阶段不取消 5 分钟轮询兜底。

## 验收标准

- 明确单条指令可直接写入 Base，并在桌面展示。
- 聊天或会议总结必须经过确认后才批量写入。
- Base 包含来源、摘要、幂等 ID 与更新时间。
- 重复确认不会创建重复记录。
- 通知正常时桌面展示延迟不超过 3 秒。
- 通知失败时 5 分钟轮询能够补偿。
- 桌面完成、改期写回同一 Base 记录。
- 写入或写回失败时均有真实、可恢复的错误状态。
