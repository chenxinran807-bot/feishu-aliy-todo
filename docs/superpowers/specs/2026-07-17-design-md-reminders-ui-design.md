# DESIGN.md 驱动的 macOS 提醒事项式界面设计

## 决策

用户在视觉对比稿中选择 B：macOS 提醒事项式。项目使用根目录 `DESIGN.md` 作为唯一视觉规范，后续界面实现不得再依赖历史 `cute / refined / minimal` 分支决定布局。

## 常驻面板

默认面板为 360 × 260px。顶部显示“今天”，副标题显示待办数量和飞书同步状态，右上角只保留新增操作。正文使用“优先处理”和“接下来”两个分组，最多展示三条任务。

任务行由圆形完成控件、单行标题和右侧元信息组成。点击标题打开详情；改期、来源、隐藏和忽略等操作放入详情，不在常驻任务行展示。

## 视觉系统

- SF Pro 系统字体。
- 白色或动态窗口材质背景。
- `#1D1D1F` 主文字，系统灰次级文字。
- `#3370FF` 为唯一交互强调色。
- `#F54A45` 仅表示逾期。
- 4px 基础间距，任务行 36px，面板圆角 18px。
- 使用细分隔线，不使用彩色任务卡。

## 数据与交互

Base 继续作为唯一任务事实源。本次只调整呈现与交互入口，不改变 15 秒飞书机器人消息检查、5 分钟 Base 兜底同步、完成写回和改期写回逻辑。

“优先处理”按现有排序结果选择最高优先级或逾期任务；剩余任务进入“接下来”。如果没有待办，显示安静的空状态，不展示统计仪表盘。

## 错误与状态

同步成功显示“飞书已同步”；离线或同步失败显示“等待飞书同步”。错误状态不占用独立横幅，不阻断本地查看。折叠和展开使用独立窗口尺寸约束，防止再次出现大窗口只显示折叠文案的问题。

## 测试与验收

- 为布局策略、任务数量限制、分组与窗口尺寸编写原生单元测试。
- 编译并打包 macOS App。
- 在真实应用窗口中检查 360 × 260px、最大尺寸和折叠状态。
- 验证界面不存在统计卡、筛选器、同步工具栏、整卡状态色、每行更多按钮和用户可见的历史名称。
- 运行现有前端测试、原生测试、类型检查与同步脚本自检。
